# Security Policy

## Supported versions

This project currently supports the latest `main` branch.

## Reporting a vulnerability

- Do not open public issues for security problems.
- Report details privately to: `mdabutalharumman@gmail.com`
- Include reproduction steps, impact, and possible fixes.

## Security notes

- Keep secrets only in environment variables.
- Never commit `.env` files.
- Rotate auth secrets if they are exposed.
